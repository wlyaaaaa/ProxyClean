## 2. 故障时间线与根本原因分析
### 阶段 I：软件崩溃与文件系统死锁
 * **症状**：应用抛出非法指令异常（0xc000001d），崩溃并进入严重的死锁状态。
 * **根本原因**：应用线程在内核态（Kernel Mode）处理 I/O 或网络 TUN 接口句柄时死锁，在内存中留下了一个**僵尸进程（Zombie Process）**。
 * **影响**：该僵尸进程死死扣住了 C:\Program Files\Clash Verge 的文件句柄，导致随后的所有安装或卸载脚本强行报错“无法终止 Clash Verge / 拒绝访问”。常规的进程强制结束命令（taskkill /F）被 Windows 内核直接拒绝。
### 阶段 II：物理清除与资源获取
 * **采取动作**：通过 PowerShell 强行清理了物理目录，并执行了**系统重启**。
 * **结果**：所有泄漏的内核句柄被彻底销毁，僵尸进程清除，物理目录锁成功释放。
 * **资源获取**：曾尝试国内镜像站（ghproxy/kkgithub）下载，但遭遇超时失败。最终通过手机海外网络成功下载官方纯净安装包 Clash_Verge_2.5.1_x64-setup.exe 并传输至电脑。

## 3. 当前网络栈状态
目前主机的网络层已通过以下命令进行了彻底的**重置与净化**：
 1. **注册表复位**：禁用了系统级 Internet 代理覆盖：
   ```powershell
   Set-ItemProperty -Path 'HKCU:\Software\Microsoft\Windows\CurrentVersion\Internet Settings' -Name ProxyEnable -Value 0
   
   ```
 2. **套接字与 IP 栈重置**：执行了 netsh winsock reset 和 netsh int ip reset，彻底剥离了第三方加速器残留的 LSP 或 TUN 钩子。
 3. **防火墙重置**：通过 netsh advfirewall reset 清空了所有乱加的拦截规则，防火墙回归官方默认状态。
> **当前网络结论**：电脑拥有 100% 健康、干净、无代理拦截的直连网络。114.114.114.114 的超时纯属外部服务器行为，**必须直接忽略**。
