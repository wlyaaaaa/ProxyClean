# Claude Code / Codex App 网络与浏览器泄露排查

本文档记录当前推荐配置和判断边界:日常使用 Claude Code / Codex App 时,优先保证
TUN/虚拟网卡路径稳定;浏览器里的 DNS/WebRTC/时区泄露排查只作为网页登录场景的补充检查。

## 结论先行

当前推荐状态:

- 代理客户端开启 TUN / 虚拟网卡 / Enhanced Mode。
- System Proxy 关闭。
- DNS 覆写关闭,除非已经验证不会影响 Claude Code / Codex App 连通性。
- 终端和项目里不配置显式代理环境变量,也不改官方 API 端点。
- 7890 / 7892 / 7897 等本地代理端口只保留一个主要客户端,避免多客户端抢路由。

如果 DNS 覆写一打开就导致 Claude Code / Codex App 连不上,应先关闭 DNS 覆写,
保留 TUN 的稳定出站路径。不要为了浏览器泄露测试结果牺牲 App 连通性。

## 为什么浏览器泄露可以暂时放低优先级

Claude Code / Codex App 日常访问的是官方服务端点,主要受以下因素影响:

- 系统底层路由是否被 TUN 正确接管。
- 本机 DNS 是否能把官方域名解析到可被 TUN 处理的地址。
- 到官方端点的 443/TLS 连接是否成功。
- 终端进程是否保持干净,没有被旧代理环境变量劫持。

浏览器泄露测试主要反映网页环境。网页可以通过 JavaScript、WebRTC、DNS 测试页读到
浏览器侧信息,例如时区、语言、WebRTC 暴露的公网候选地址、DNS 解析器分布等。
这些结果对网页登录有意义,但不等于 Claude Code / Codex App 本体一定暴露同样信息。

因此,如果只是用浏览器完成登录,不长期在网页版里工作,浏览器 DNS/WebRTC 泄露的优先级
低于 App 的稳定连通性。

## 登录网页时需要知道的边界

网页登录仍然是浏览器行为。登录页面可能看到:

- 浏览器出口 IP。
- 浏览器时区和系统时间。
- 浏览器语言、地区、User-Agent 等常规指纹。
- WebRTC 暴露的候选公网地址。
- DNS 测试页观测到的解析器位置。

这些属于网页登录侧的浏览器环境信号。它们不能通过给 Claude Code 终端配置代理环境变量来解决,
也不建议用会破坏 App 连通性的 DNS 覆写硬压。

## DNS 覆写的当前判断

本机实测现象:

- DNS 覆写关闭时,Claude Code / Codex App 路径可用。
- DNS 覆写打开后,浏览器可能仍能访问部分网站,但 Claude Code / Codex App 可能连不上。

这通常说明 DNS 覆写规则让非浏览器进程的解析或 TUN 路由变得不稳定。常见高风险组合包括:

- `redir-host`
- IPv6 开启但节点和规则没有完整支持
- `system` DNS 和公共 DNS 混用
- 国内 DNS 与海外 DoH 混用
- 复杂 fallback / nameserver-policy 规则

排查原则是先稳定,再收紧。当前应保持 DNS 覆写关闭。如果以后要重新测试 DNS 覆写,
先使用最小配置,每次只改一个变量,确认 Claude Code / Codex App 能连后再继续。

## 推荐检查顺序

1. 双击 `查看当前走哪个.bat`,先看顶部结论。
2. 确认 System Proxy 是关闭状态。
3. 确认环境变量代理为空。
4. 确认只有目标代理端口在监听,例如只保留 7897 或只保留 7892。
5. 确认默认出口和强制目标端口出口一致。
6. 如果 App 可用,不要继续为浏览器 DNS 泄露强开 DNS 覆写。

## 合规与卫生原则

- 不在项目、终端启动脚本、shell profile 或配置文件中写入本地代理端口。
- 不把第三方中转端点写进项目配置。
- 不提交密钥、账号、节点订阅、机场配置、个人端口策略。
- 与网络相关的本地临时适配放在 `.gitignore` 覆盖的私有文件里。
- 本项目脚本只负责清理死端口、残留系统代理、残留环境变量和孤儿路由,不负责绕过服务规则。

## 什么时候再处理浏览器泄露

只有在以下情况才值得单独处理浏览器侧:

- 长期使用网页版 Claude 或其他网页服务。
- 登录频繁失败,且已经确认 App/TUN 路径稳定。
- 需要让浏览器 WebRTC 不暴露非页面出口的公网地址。
- 需要让浏览器时区、地区、语言和出口地区保持一致。

这类处理应只作用于浏览器本身,例如浏览器配置、扩展或浏览器策略。不要把问题扩散到
Claude Code / Codex App 的终端环境。
